Object Detail

Workspace

The Workspace is the main musical working scope inside a Chord Lab project. It is where progression, Beat Lane, generated patterns, harmonic context, and most editing operations actually live. This object matters because many interactions that feel "project-wide" to the user are really workspace-scoped in the model.

Definition

A Workspace is a contained musical document scope inside the project. It owns the progression, Beat Lane arrangement, generated pattern library, and workspace-level musical settings such as key and scale context. When the user edits chords, places beats, or designs patterns, they are usually working inside one selected workspace.

Primary work scope Model object Owns arrangement state Workspace-local truth
Key distinction:
The Workspace is not just a UI tab or editor context. It is a real model boundary that owns most of the musical state the user edits.

What This Object Is Not

Is This Object

The object that contains the active progression, Beat Lane, and generated patterns for one musical context.

Not This Object

The full project root, or a single sub-object like a progression or Beat Lane block. Those sit above or below the workspace.

Boundary Case

The editor may show one selected workspace at a time, but that visible selection is separate from the underlying workspace object itself.

Relationships In Chord Lab

Related Object Relationship Direction Notes
Project Workspace is contained by the project root Upstream The project defines document scope; the workspace defines active musical working scope.
Progression Workspace owns the chord arrangement for this scope Downstream Progression is the main harmony/timeline object inside the workspace.
Beat Lane Workspace owns the placed beat arrangement for this scope Downstream Beat Lane is the source of truth for placed beats in the workspace.
Generated Pattern Workspace owns the reusable beat asset library Downstream Patterns are shared within a workspace and referenced by Beat Lane blocks.
Editor Editor usually presents one selected workspace at a time Downstream consumer A lot of editor refresh work is really about reflecting the current workspace cleanly.

Mutation Touch Points

  • Selection mutations often retarget which workspace the editor is currently consuming.
  • Value mutations for progression, Beat Lane, and generated patterns are usually workspace-local before they are project-wide.
  • Beat Lane and generated pattern mutations meet here, because both objects live inside the workspace and interact through shared references.

Design Tensions / Failure Modes

  • It is easy for code to confuse project-wide and workspace-local scope, which makes update contracts broader and more expensive than they need to be.
  • If workspace-local edits are routed through project-wide undo/publish machinery by default, the UI can feel brittle even when the conceptual change is small.
  • The workspace owns both chord and beat domains, so it is the natural place where Chord Roll, Beat Lane, Pattern Generator, and playback truth intersect.
