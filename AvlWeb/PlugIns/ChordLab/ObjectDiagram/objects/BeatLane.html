Object Detail

Beat Lane

The Beat Lane is the workspace-owned arrangement object for placed beats. It is the source of truth for where reusable beat patterns are placed in musical time inside a workspace. This object matters because Pattern Generator designs patterns, but Beat Lane determines when and where those patterns actually exist in the arrangement.

Definition

The Beat Lane is the model object that owns placed beat references for a workspace. It does not own the reusable beat pattern assets themselves; instead, it owns the arrangement of placed beat blocks that reference those assets over time. When the user drops a pattern into the timeline, moves a beat, or opens a placed beat for editing, they are interacting with Beat Lane truth.

Placed beat truth Workspace-owned Arrangement object Pattern-reference container
Key distinction:
The Beat Lane is not the same as Generated Pattern. A generated pattern is a reusable asset; the Beat Lane is the arrangement object that decides where that asset is placed.

What This Object Is Not

Is This Object

The workspace-level timeline object that contains placed beat blocks and defines real beat placement in the arrangement.

Not This Object

A generated pattern asset or the Pattern Generator session. Those design or edit reusable content, but they are not the arrangement source of truth.

Boundary Case

Double-clicking a placed beat can open the Pattern Generator, but the Beat Lane still remains the authority for whether that beat is actually placed in the arrangement and when it plays.

Beat Lane Diagram

The Beat Lane sits under Workspace, references generated patterns through placed beat blocks, and drives arrangement-following beat behavior in playback and editing.

Mutation Touch Points

  • Beat Lane mutations add, move, resize, retarget, and delete placed beat blocks under this object.
  • Selection mutations can target Beat Lane blocks, but that targeting is separate from the Beat Lane’s arrangement truth.
  • Generated pattern mutations can affect what existing Beat Lane placements sound like because the Beat Lane references shared pattern assets.

Design Tensions / Failure Modes

  • The Beat Lane is conceptually the source of truth for placed beats, but some current paths appear to under-propagate that truth until a later stronger action occurs.
  • If moving a beat makes other stale state suddenly become correct, that suggests the Beat Lane contract path is not yet invoked consistently across all beat-related operations.
  • Because the Beat Lane sits between shared pattern assets and real arrangement use, weak boundaries here make Pattern Generator behavior feel especially brittle.
