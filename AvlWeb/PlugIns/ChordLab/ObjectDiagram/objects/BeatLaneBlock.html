Object Detail

Beat Lane Block

A Beat Lane Block is the atomic placed beat reference inside the Beat Lane. It lives under Beat Lane arrangement state and points at a generated pattern asset while also carrying placement-specific timing and scope. This object matters because it is the exact boundary where reusable pattern design becomes a real placed event in the workspace arrangement.

Definition

A Beat Lane Block is the placement object that sits on the Beat Lane timeline and references a Generated Pattern for musical content. It carries arrangement-local information such as start position, extent, and asset reference, making it the concrete representation of a beat used in the workspace. When the user places a pattern in the Beat Lane, drags it to another bar, resizes it, or double-clicks it for placed-beat editing, they are interacting with Beat Lane Block state.

Atomic placement Beat Lane-owned Pattern reference Arrangement-local timing
Key distinction:
A Beat Lane Block is not the same as a Generated Pattern. The block is a placed reference with arrangement timing; the generated pattern is the reusable asset it points to.

What This Object Is Not

Is This Object

The specific placed beat instance the user can move, resize, select, and open from the Beat Lane timeline.

Not This Object

The Beat Lane container as a whole or the shared Generated Pattern asset. It is the per-placement object under the lane.

Boundary Case

Opening a Beat Lane Block in the Pattern Generator may expose the shared generated pattern for editing, but the block still remains the arrangement-specific context that defines when that pattern is actually used.

Mutation Touch Points

  • Beat Lane mutations add, move, resize, retarget, and delete Beat Lane Blocks as placement objects under the lane.
  • Selection mutations can target a Beat Lane Block directly, making it the current arrangement edit target without changing the shared pattern asset.
  • Generated pattern mutations can change what a Beat Lane Block sounds like indirectly because the block references shared pattern content.

Design Tensions / Failure Modes

  • If Beat Lane Block timing changes do not fully propagate, arrangement-following cursor and playback behavior can look stale until a later stronger refresh repairs them.
  • The same placed block can imply both arrangement-local truth and shared-asset editing, which makes boundaries blurry if contracts are not explicit.
  • Because the block is the point where reuse meets placement, weak ownership rules here can make Pattern Generator interactions feel especially fragile.
