Object Detail

Pattern Generator Session

A Pattern Generator Session is the tool-local state object that defines how the Pattern Generator is currently operating. It lives on the runtime/UI side rather than in the shared workspace model and carries mode, local target references, local selection, and preview behavior for one open editing context. This object matters because many of Chord Lab’s current brittleness issues come from mixing session state with shared Generated Pattern state and Beat Lane placement truth.

Definition

A Pattern Generator Session is the current working context of the Pattern Generator tool. It owns local mode state such as standalone design mode vs placed-beat mode, local references such as the opened Beat Lane Block or targeted Generated Pattern, and transient UI state such as selected step or blank-session intent. When the user opens the tool, switches its mode, changes local target context, or interacts with the session-level selection inside the grid, they are mutating Pattern Generator Session state.

Tool-local state Runtime/UI object Mode-sensitive Preview policy carrier
Key distinction:
Pattern Generator Session is not the same as Generated Pattern. The session describes how the tool is currently operating; the generated pattern is the shared asset the session may be editing.

What This Object Is Not

Is This Object

The local editing context that decides whether the tool is in design mode, placed-beat mode, what it is targeting, and how its cursor/preview should behave.

Not This Object

The shared generated pattern asset or the Beat Lane Block model object. Those are model-side truths that the session references.

Boundary Case

Clicking a step in the grid can change local step selection inside the session without changing the shared pattern asset itself, unless the user also edits a value or toggles the step.

Relationships In Chord Lab

Related Object Relationship Direction Notes

Mutation Touch Points

  • Session mutations open, close, retarget, or reset the Pattern Generator Session without necessarily changing shared musical data.
  • Selection mutations can change local selected-step state inside the session even when the generated pattern asset stays unchanged.
  • Value mutations may be initiated through the session, but they should write into Generated Pattern or Beat Lane state rather than becoming trapped as session-only truth.

Design Tensions / Failure Modes

  • If session state and generated-pattern state are not clearly separated, the Pattern Generator can appear to mutate the wrong thing or refresh too broadly.
  • If placed-beat context is not represented explicitly in the session, the tool can drift into dishonest behavior where cursor or playback no longer match arrangement truth.
  • If local preview behavior is wired through heavy processor-wide refreshes instead of through a scoped session contract, the tool feels laggy and fragile.
