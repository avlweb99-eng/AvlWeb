Object Detail

Generated Pattern

A Generated Pattern is the reusable beat asset that Pattern Generator edits and Beat Lane blocks reference. It lives under a workspace as shared pattern-library state rather than as a placed arrangement event. This object matters because many current Chord Lab behaviors depend on keeping the reusable asset cleanly separated from the placed Beat Lane truth and from local tool-session state.

Definition

A Generated Pattern is the model object that stores reusable beat content such as step placement, step values, bar count, and pattern-level identity. It is owned by a workspace pattern library and can be referenced by zero, one, or many Beat Lane blocks. When the user creates a new beat design, renames a pattern, or edits steps in the Pattern Generator, they are mutating Generated Pattern state.

Reusable beat asset Workspace-owned Shared library state Pattern Generator target
Key distinction:
Generated Pattern is not the same as Beat Lane. The pattern is a reusable asset definition; Beat Lane is the arrangement object that decides where that asset is actually placed.

What This Object Is Not

Is This Object

The shared beat asset that stores steps, step values, and pattern identity independently of any single placement in the arrangement.

Not This Object

A placed Beat Lane block. Placement timing and arrangement truth live in Beat Lane, not in the generated pattern asset.

Boundary Case

The Pattern Generator may open on a placed Beat Lane block, but the dialog is still editing the shared generated pattern unless and until the product introduces separate copy-on-edit semantics.

Generated Pattern Diagram

Generated Pattern sits between workspace-owned library state and the consumers that either place it in the arrangement or edit/audition it locally.

Mutation Touch Points

  • Generated pattern mutations create, rename, edit, and delete this asset from the workspace library.
  • Session mutations can change which generated pattern a Pattern Generator session is targeting without changing the asset itself.
  • Beat Lane mutations can change where a generated pattern is used in the arrangement without mutating the pattern asset content.

Design Tensions / Failure Modes

  • The biggest current tension is keeping generated pattern asset edits lightweight while still propagating correctly to all placed Beat Lane references.
  • If a placed-beat edit path and a design-mode edit path do not clearly share the same asset contract, the Pattern Generator can feel brittle or misleading.
  • Because one asset may be referenced in several places, delete, rename, and shared-edit semantics need to remain explicit or users can lose trust quickly.
