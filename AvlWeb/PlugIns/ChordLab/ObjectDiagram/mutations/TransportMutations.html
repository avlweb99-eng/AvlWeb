Transport Mutations

Transport mutations are state changes that alter playback authority, play/stop state, or the active transport timeline. They determine whether Chord Lab is following the host, using internal transport, or applying a tool-local audition policy on top of that authority. This distinction matters because transport is where musical truth becomes audible and visible, and conflicting authority rules are a direct source of untrustworthy behavior.

Definition

A transport mutation happens when playback state or transport authority changes. This includes host playback starting or stopping, internal transport starting or stopping, and any change that switches which timeline the UI and playback engine must follow. The core question on this page is: when transport state changes, how do we keep playback authority explicit and prevent host, internal, and tool-local behaviors from competing?

Playback authority Time-domain change Host-sensitive Must stay explicit
Key distinction:
A transport mutation changes who is driving playback or where playback time comes from. It is different from a preview-layer update, a session change, or an ordinary cursor repaint.

What This Is Not

Is Transport Mutation

Host starts playing, internal transport starts, or the active playback authority switches from internal to host.

Not Transport Mutation

Repainting the cursor on the next timer tick. That is a visual update consuming transport state, not a transport mutation by itself.

Boundary Case

Turning on Pattern Generator auto-audition can affect local playback behavior, but it does not become a transport mutation unless transport authority or play state itself changes.

Transport Mutation Diagram

Transport mutations start from authority or play-state changes, rewrite active playback policy, and then retarget every consumer that depends on the current time source.

Examples

Host Takes Over

Internal transport is running, then the host starts playing. Host authority must immediately replace internal authority, and internal controls should become disabled.

Internal Audition Mode

The host is idle and the user presses play in Chord Lab. Internal transport becomes authoritative and the arrangement plays under Chord Lab timing.

Tool-Local Loop Under Authority

Pattern Generator design mode loops locally while respecting the fact that host transport, if active, still owns the main playback authority.

Non-Examples

  • Repainting the playback cursor each timer tick This is a consumer update driven by transport state, not a transport mutation.
  • Opening the Pattern Generator This is a session mutation unless transport authority or play-state changes with it.
  • Editing a Beat Lane block This is a Beat Lane mutation, though its effects are rendered under transport.
  • Changing tempo or loop length These are value mutations that transport consumes.

Transport Mutation Contract

Mutation Authoritative Change Must Update Must Not Update Notes
Host starts playing Host becomes the active playback authority Playback engine source of time, transport status, internal button enabled state, arrangement cursors, relevant preview policy Stored musical data, unrelated layout state, unrelated selections Host always wins. This is an explicit product requirement, not just an implementation detail.
Internal transport starts or stops Chord Lab internal authority becomes active or inactive when host is idle Playback engine, transport buttons, transport labels, arrangement cursors, internal audition behavior Host authority if host is already playing, unrelated model data, generic editor rebuild Internal transport is first-class, but only when host is not playing.
Authority fallback or swap The active time source changes between host, internal, or stopped state Any view or tool that depends on current playback time or authority policy Pattern assets, Beat Lane model data, unrelated session-local selections Authority changes are where overlap bugs and “double playback” bugs are most likely if the contract is unclear.

Transport Mutation Types In Chord Lab

Mutation Category What Changes Notes
Host playback begins Authority mutation Host transport becomes authoritative This should immediately suppress internal transport intent and disable internal controls.
Host playback ends Authority mutation Host authority ends and transport may fall back to internal or stopped state The fallback policy must be explicit and easy to understand.
Internal play Play-state mutation Chord Lab internal transport becomes active This is valid only when the host is not playing.
Internal stop Play-state mutation Chord Lab internal transport becomes inactive This should cleanly stop internal arrangement playback without corrupting tool-local context.
Preview / audition policy retarget Transport-adjacent policy mutation Local playback behavior changes to respect the active transport authority This sits at the boundary between transport and session behavior.
Cursor authority switch Transport consumer consequence Chord Roll, Beat Lane, and Pattern Generator cursor logic follow a different time source Often driven by a transport mutation rather than being the mutation itself.

Design Tensions / Failure Modes

  • Host authority is implemented in the engine, but the UI has not always made that authority explicit enough, which makes transport feel inconsistent.
  • Internal transport, preview, and tool-local audition behavior can become entangled if they are all treated as one generic playback system.
  • Pattern Generator and placed-beat editing expose transport design conflicts quickly because they depend on both global authority and local behavior policy.

Open Questions

  • Exactly which transport-facing UI controls should become disabled versus merely passive when host playback is active?
  • How should future custom loop-region features fit into the authority model without making cursor ownership ambiguous?
  • Where should tool-local audition policy live so it can respect transport authority without being confused for a second transport?
