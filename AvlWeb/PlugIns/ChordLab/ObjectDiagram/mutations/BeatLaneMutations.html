Beat Lane Mutations

Beat Lane mutations are state changes that add, remove, move, resize, or retarget placed beat blocks in the arrangement. They change arrangement truth inside the workspace by altering where and how generated patterns are used in musical time. This distinction matters because Beat Lane is the source of truth for placed beats in the progression, so its mutations must update both arrangement-facing UI and playback behavior correctly and immediately.

Definition

A Beat Lane mutation happens when the arrangement of placed beat blocks changes. This includes creating a Beat Lane block, moving it, changing its timing or referenced pattern, and deleting it from the workspace. The core question on this page is: when arrangement truth changes in Beat Lane, which consumers must update together so the UI and playback remain trustworthy?

Arrangement truth Placed reference change Playback-sensitive Should be deterministic
Key distinction:
A Beat Lane mutation changes placed arrangement data. It is different from a generated pattern mutation, which changes the reusable asset, and different from a Beat Lane selection mutation, which only changes the current target block.

What This Is Not

Is Beat Lane Mutation

Place a pattern into Beat Lane, move an existing Beat Lane block to a new bar, or delete a placed beat from the arrangement.

Not Beat Lane Mutation

Editing a step velocity in the Pattern Generator. That changes the pattern asset, not the placement itself.

Boundary Case

Double-clicking a Beat Lane block to open the Pattern Generator is primarily a session mutation. It becomes a Beat Lane mutation only when the placed block itself changes.

Beat Lane Mutation Diagram

Beat Lane mutations start from arrangement edits, rewrite placed beat-block truth, and must keep the Beat Lane view, playback interpretation, and any open placed-beat session aligned.

Examples

Place Pattern Into Beat Lane

Drop a generated pattern onto bar 3. Beat Lane arrangement truth changes even though the pattern asset itself stays the same.

Move Beat Block

Drag a Beat Lane block from bar 2 to bar 5. The block’s placement changes, so cursor activation and playback windows should move with it immediately.

Retarget Beat Block

Swap a placed block so it points to a different generated pattern. Arrangement truth and pattern reference edges both change at once.

Non-Examples

  • Clicking a Beat Lane block so it becomes the current target This is selection unless the block placement itself changes.
  • Editing a step value inside the referenced pattern This is a generated pattern mutation, even though Beat Lane placements may later reflect it.
  • Opening Pattern Generator on a placed beat This is a session mutation unless the block is also modified.
  • Starting or stopping transport These are transport mutations.

Beat Lane Mutation Contract

Mutation Authoritative Change Must Update Must Not Update Notes
Place Beat Lane block A new placed block is added to the Beat Lane arrangement Beat Lane view, placement counts, playback windows, any arrangement-following session that cares about the new block Unrelated pattern asset contents, unrelated editor tabs, generic full layout This is where a reusable pattern becomes arrangement truth at a specific location.
Move or resize Beat Lane block Existing block timing or extent changes in Beat Lane Beat Lane visuals, playback timing/cursor logic, placed-beat Pattern Generator context if open on that block Unrelated workspaces, unrelated generated patterns, unrelated local sessions This is the path that appears to act like a “repair refresh” in some current behaviors, which means its contract is probably stronger than other paths.
Delete Beat Lane block Placed reference is removed from the arrangement Beat Lane visuals, playback windows, placement counts, any open session that was tied to the removed block Generated pattern asset contents, unrelated transport state, unrelated selections Deletion changes arrangement truth, but should not be mistaken for deleting the underlying generated pattern asset.

Beat Lane Mutation Types In Chord Lab

Mutation Category What Changes Notes
Add Beat Lane block Arrangement mutation A placed beat reference is inserted into the workspace timeline This is the main entry point from pattern design into arrangement use.
Move Beat Lane block Arrangement mutation Start position of the placed block changes This should immediately retime both visuals and playback behavior.
Resize or change duration/extent Arrangement mutation How long the placed Beat Lane reference occupies the arrangement changes This matters directly for cursor activation and arrangement-following behavior.
Retarget Beat Lane block Arrangement-reference mutation Placed block points at a different generated pattern asset This is the clearest link between Beat Lane and generated pattern domains.
Delete Beat Lane block Arrangement mutation Placement is removed while the reusable pattern may still remain in the library Deleting placement and deleting pattern must stay clearly separate.
Rearrange multiple beats Arrangement mutation Several placements change in a single operation This appears to be a “good path” in current behavior because it often causes the rest of the system to become correct.

Design Tensions / Failure Modes

  • Beat Lane appears to be the source of truth for placed beats, but some operations do not fully propagate that truth until a later stronger action occurs.
  • If moving or rearranging beats “fixes” stale state elsewhere, that suggests some Beat Lane mutation paths are invoking a more complete update contract than others.
  • Placed-beat Pattern Generator sessions are especially sensitive because they must follow arrangement truth rather than free-running local behavior.

Open Questions
